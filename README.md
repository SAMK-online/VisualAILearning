# Visual Learning Platform

**Learn Anything Visually - Powered by AI**

An AI-powered interactive educational platform that transforms any topic into beautiful, engaging visualizations with step-by-step animations. Built with modern web technologies and powered by OpenAI GPT-4 or Anthropic Claude.

## ğŸš€ Quick Start

### Prerequisites
- **Python 3.9+** (for backend)
- **Node.js 18+** (for frontend)
- **OpenAI API Key** or **Anthropic API Key**

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd FidexLovable
```

### 2. Backend Setup

```bash
# Navigate to backend
cd backend

# Create virtual environment
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
cp .env.example .env
# Edit .env and add your API key:
# OPENAI_API_KEY=sk-your-key-here
# OR
# ANTHROPIC_API_KEY=sk-ant-your-key-here

# Run the backend server
python run.py
```

The backend API will be available at `http://localhost:8000`

**API Documentation**: http://localhost:8000/api/docs

### 3. Frontend Setup

```bash
# Open a new terminal and navigate to frontend
cd frontend

# Install dependencies
npm install

# Create environment file (optional - uses localhost:8000 by default)
cp .env.example .env

# Run the development server
npm run dev
```

The frontend will be available at `http://localhost:5173`

### API Endpoints

#### POST `/api/generate-visualization`
Generate a visualization for a topic.

**Request:**
```json
{
  "topic": "Binary Search Tree"
}
```

**Response:**
```json
{
  "success": true,
  "topic": "Binary Search Tree",
  "title": "Binary Search Tree Visualization",
  "description": "A binary search tree is...",
  "visualization_type": "tree",
  "components": [...],
  "steps": [...],
  "interactive_elements": [...],
  "metadata": {...}
}
```

#### GET `/api/examples`
Get example topics organized by category.

#### GET `/api/bst-demo`
Serve the interactive Binary Search Tree demo HTML page (for demonstration purposes).

#### GET `/health`
Health check endpoint.

## ğŸ¨ Customization

### Adding New Visualization Types

1. **Update Backend** (`backend/app/models.py`):
```python
visualization_type: Literal["tree", "graph", "flowchart", "your_new_type"]
```

2. **Update AI Prompt** (`backend/app/ai_service.py`):
Add your new visualization type to the system prompt.

3. **Update Frontend Renderer** (`frontend/src/components/VisualizationCanvas.tsx`):
```typescript
case "your_new_type":
  renderYourNewType(ctx, data, currentStep, dimensions);
  break;
```

### Configuring AI Model

Edit `backend/.env`:
```env
# Use OpenAI
AI_PROVIDER=openai
OPENAI_MODEL=gpt-4

# OR use Anthropic Claude
AI_PROVIDER=anthropic
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022
```

## ğŸ› Troubleshooting

### Backend Issues

**Port 8000 already in use:**
```bash
# Change port in backend/.env
PORT=8001
```

**AI API errors:**
- Verify your API key is correct
- Check you have credits/quota remaining
- Ensure correct model name

### Frontend Issues

**API connection failed:**
- Verify backend is running on port 8000
- Check `VITE_API_BASE_URL` in `frontend/.env`

**Build errors:**
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```
